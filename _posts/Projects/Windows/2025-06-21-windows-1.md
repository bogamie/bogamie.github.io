&nbsp;&nbsp; 컴퓨터는 외부 세계와의 입력 및 출력을 제공하는 다양한 장치들로 구성되어 있다. 일반적인 장치로는 키보드, 마우스, 오디오 컨트롤러, 비디오 컨트롤러, 디스크 드라이버, 네트워크 포트 등이 있다. 디바이스 드라이버는 운영체제와 장치 사이의 소프트웨어 연결을 제공한다. 이러한 이유로 I/O는 디바이스 드라이버 개발자에게 매우 중요하다.

&nbsp;&nbsp; Windows 커널 모드 I/O 관리자는 애플리케이션과 디바이스 드라이버가 제공하는 인터페이스 간의 통신을 관리한다. 디바이스는 운영체제가 따라가지 못할 정도의 속도로 작동할 수 있기 때문에, 운영체제와 디바이스 드라이버 간의 통신은 주로 I/O 요청 패킷(IRP, I/O Request Packet)을 통해 이루어진다. 이 패킷들은 네트워크 패킷이나 Windows 메시지 패킷과 유사하며, 운영체제에서 특정 드라이버로, 그리고 한 드라이버에서 다른 드라이버로 전달된다.

&nbsp;&nbsp; Windows I/O 시스템은 스택(stacks)이라고 불리는 계층형 드라이버 모델을 제공한다. 일반적으로 IRP는 통신을 용이하게 하기 위해 동일한 스택 내의 한 드라이버에서 다른 드라이버로 전달된다. 예를 들어, 조이스틱 드라이버는 USB 허브와 통신해야 하고, USB 허브는 USB 호스트 컨트롤러와 통신해야 하고, USB 호스트 컨트롤러는 PCI 버스를 통해 컴퓨터 하드웨어의 나머지 부분과 통신해야 한다. 이 스택은 조이스틱 드라이버, USB 허브, USB 호스트 컨트롤러와 PCI 버스로 구성된다. 이 통신은 스택 내의 각 드라이버가 IRP를 송수신함으로써 조정된다.

&nbsp;&nbsp; 드라이버가 스택 내에서 효율적으로 동작하기 위해 IRP를 제시간에 송수신하는 것이 매우 중요하다. 만약 드라이버가 스택의 일부분임에도 불구하고 IRP를 제대로 수신, 처리, 전달하지 못하면 시스템 충돌을 일으킬 수 있다.
